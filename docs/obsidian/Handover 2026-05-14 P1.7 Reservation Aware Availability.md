## Behaviour Now

Public event detail:

- `GET /api/public/events/[eventId]` still returns `{ event }`.
- Event must be `published`.
- Each ticket type includes both:
  - `quantity`: raw remaining inventory as stored on `TicketType`.
  - `availableQuantity`: public reservation-aware availability.
- `availableQuantity` is computed as:

```text
max(0, quantity - activeReservedQuantity)
```

Reservation counting:

- Only reservations with `status = active` and `expiresAt > now` reduce public availability.
- Expired active reservations do not reduce public availability.
- `released`, `confirmed`, and `expired` reservation statuses do not reduce public availability.
- Over-reserved edge cases clamp availability to `0`.
- Multiple ticket types are calculated independently.

Public purchase UI:

- Uses `availableQuantity` for visible remaining count.
- Uses `availableQuantity` for sold-out state.
- Uses `availableQuantity` for quantity input `max`.
- Uses `availableQuantity` for client validation.
- Uses `availableQuantity` for buy-button disabled state.

Read-only guarantee:

- Public event detail does not run stale cleanup.
- Public event detail does not expire reservations.
- Public event detail does not create demo data.
- Public event detail does not write any database rows.

## Implementation Notes

Primary files:

- `lib/events/public-events.ts`
- `app/api/public/events/[eventId]/route.ts`
- `app/(public)/events/[eventId]/page.tsx`
- `components/events/public-ticket-purchase.tsx`

The detail route is thin and delegates to `getPublishedEventDetail()`.

Availability mapping lives in `lib/events/public-events.ts` and uses one grouped query:

```ts
ticketReservation.groupBy({
  by: ["ticketTypeId"],
  where: {
    ticketTypeId: { in: ticketTypeIds },
    status: "active",
    expiresAt: { gt: now }
  },
  _sum: { quantity: true }
})
```

`_sum.quantity` is null-safe and defaults to `0`.
